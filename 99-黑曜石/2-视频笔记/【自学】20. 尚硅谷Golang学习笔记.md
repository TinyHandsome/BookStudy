## 1. Golang开山篇

1. go的优势：

   - 数据处理、处理大并发
   - 区块链研发工程师
   - 分布式/云计算 软件工程师
   - c#是和java一桌的，c/c++和go这才是一桌，其他脚本语言一桌

2. CDN：内容分发网络

3. golang的吉祥物：在2009年的开源发布会上，Renee（发明人之一罗布派克的老婆，Renee French）建议改编 WFMU 的gopher作为吉祥物，该地鼠没有名称，仅被称为“Go gopher”。

   ![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/c4c47ce575004c268076c9a7ace35555.png)

4. 程序：完成某个功能的指令的集合

5. go的历史：

   - go是谷歌的亲儿子
   - go为了解决的问题：即使使用大型编译集群之上进行一次 build，**时间周期**也是不可被我们接受的
     - 硬件技术更新频繁，软件落后硬件，不能发挥 **多核多CPU** 的优势
     - 缺乏一个足够简洁高效的编程语言，风格不统一、计算能力不够、处理大并发不好
     - c/c++ 编译速度慢，存在内存泄漏等问题
   - [go的创作者](https://zhuanlan.zhihu.com/p/322560760)：肯·汤普逊（Ken Thompson），罗布·派克（Rob Pike），以及 罗伯特·格瑞史莫（Robert Griesemer）
     - 肯·汤普逊：1966年创造了b语言，继而用一个月创造了UNIX操作系统；1971年和丹尼斯·利奇（Dennis Ritchie）一起共同发明了C语言，在2年后用c语言重写了UNIX
     - 罗布·派克：不仅参与UNIX开发，还是UTF-8的两个发明人之一，著作：**《UNIX环境编程》**（The Unix Programming Environment）和**《程序设计实践》** （The practice of programming）
     - 罗伯特·格瑞史莫：参与js/v8引擎的设计 和 java hotspot 虚拟机的研发

6. go语言的特点

   - 既能达到静态编译语言的安全和性能，又达到了动态语言开发维护的高效率

   - go = c + python

   - go保留了跟c一样的编译执行方式以及弱化的指针

     ```go
     func testPtr(num *int) {
     	*num = 20
     }
     ```

   - 引入包的概念，用于组织程序结构，go语言的一个文件都要归属于一个包，而不能单独存在

   - 垃圾回收机制，内存自动回收，不需要开发人员管理，避免出现内存泄漏

   - 天然的并发

     - 从语言层面支持并发，实现简单
     - goroutine，轻量级线程，可实现大并发处理，高效利用多核
     - 基于CPS（Communicating Sequential Processes）并发模型实现

   - 吸收了管道通信机制，形成Go语言特有的管道channel，通过管道channel，可以实现不同的goroute之间的相互通信

   - 函数可以返回多个值

   - go语句后面不用带分号

   - 新的创新：切片slice、延时执行defer等

7. 环境搭建

## 2. go语言入门

1. go程序开发基础

   - 项目路径

     ```
     Project
     	src
     		go_code
     			project01
     				main
     					hello.go
     				package
     ```

   - 构建：`go build hello.go`

   - 执行：`hello.exe`，简易执行：`go run hello.go`，这个直接把编译和运行放到一步了

   - linux下运行：

     - build之后是 `hello`
     - 执行：`./hello`

2. golang执行流程分析

   - 如果编译了，那么直接可以拷贝编译后的文件到没有go开发环境的机器上运行
   - 编译时会把依赖的库文件放在可执行文件中，所以会大不少

3. 编译和运行说明

   - `go build -o myhello.exe hello.go`：编译输出指定文件名，如果不是windows，则是可执行的二进制文件
   - 如果编译出错，编译时，会在错误的那一行报错
   - go应用程序的入口是 `main()` 函数
   - go语言严格区分大小写
   - go语言可以用分号隔开多行写一行，但是不建议
   - go语言定义的变量或者import的包没有使用到，无法通过编译

4. go语言的转义字符

   - `\t、\n、\\、\"、\r`：制表符、换行、斜杠、引号、一个回车，只把光标移到头，会把后面的内容顶头再输出

5. 注释

   - 行注释：`//`  :heavy_check_mark: 推荐
   - 块注释：`/*     */`

6. 代码风格：使用gofmt来格式化 `gofmt -w main.go`，格式化并写入

7. go要求左大括号必须跟在函数定义行

8. go语言官方编程指南，国内中文网：https://studygolang.com/，标准文档库：https://studygolang.com/pkgdoc

9. dos基础

   - `dir`：查看当前目录
   - `cd /d f:`：切换到f盘
   - `cd d:\xxx`：绝对路径切换
   - `cd ..`：切换到上一级
   - `md xxx`：新建目录
   - `rd /q/s xxx`：删除目录，/q不用询问，/s同时删除子目录
   - `echo hellomytext > d:\as\asd\abc.txt`：新建或追加内容到文件
   - `copy xxx.txt d:\a\b\yyy.txt`：复制文件，并重命名，不写文件名就是直接复制
   - `move xxx.txt d:\`：剪切、移动
   - `del aa.txt`、`del *.txt`：删除文件
   - `cls`：清屏
   - `exit`：退出dos

## 3. 变量和运算符

1. 变量的使用步骤：声明（定义）、赋值、使用

2. go变量使用的三种方式：

   1. 指定变量类型，声明后如果不赋值，将会使用默认值：`var i in`
   2. 根据值自行判定变量类型，类型推导：`var a = 10`
   3. 省略var，`:=` 左侧的变量不应该是已经声明过的，否则会导致编译错误

3. **多变量声明**：

   1. `var n1, n2, n3 int`
   2. `var n4, n5, n6 = 100, "tom", true`
   3. `n7, n8, n9 := 100, "tom", true`

4. 全局变量

   - 注意：全局变量不能用 `:=` 的写法，只能用var

     ```go
     var a1, a2, a3 = 1, 2, 3
     var (
     	b1 = 10
     	b2 = 20
     	b3 = 39
     )
     ```

5. 该区域的数据值可以在**同一类型**范围内不断变化

6. 变量在同一个作用于（函数、代码块）内不能重名

7. 变量 = 变量名 + 值 + 数据类型

8. “+”：数值型就是加法，字符串就是拼接

9. **数据类型**

   - **基本数据类型**
     - 数值型
       - 整数类型：int、int8、int16、int32、int64、uint、uint8、uint16、uint32、uint64、byte
       - 浮点类型：float32、float64
     - 字符型：没有专门的字符型，使用byte来保存单个字母字符
     - 布尔型：bool
     - 字符串：string，官方将string归属到基本数据类型，在java中string是一个类
     - 复数类型：complex64、complex128
     - rune：int32的别名，表示一个Unicode码点
   - **派生/复杂数据类型**
     - 指针 Pointer
     - 数组
     - 结构体 struct，类似别的语言的class
     - 管道 Channel
     - 函数
     - 切片 slice，类似动态数组
     - 接口 interface
     - map，类似集合

11. 浮点类型

    ![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/c022062a89094605ab9a27711d846865.png)

    1. golang的浮点类型有固定的范围和字段长度，不受具体OS的影响
    1. golang的浮点型默认声明为float64，`var num = 1.1`
    1. 浮点数都是有符号位的，浮点数=符号位+指数位+尾数位
    1. 尾数部分可能丢失，造成精度损失
    1. 支持小数的科学计数法输入：`num2 := 5.1234e2`，大写E和小写e都行，**开发中推荐使用float64**

12. 字符类型 char

    1. golang 没有专门的字符类型，如果要存储单个字符（字母），一般用byte来保存
    2. 对于传统的字符串是由字符组成的，而 **go的字符串是由字节组成的**
    3. 输出 byte 就是直接输出对应的十进制（ASCII码），不过可以通过格式化输出来输出字符的值：`fmt.Printf('c1=%c, c2=%c', c1, c2)`，如果字符超出了byte的大小 >255，比如中文汉字，则需要用int才能存
    4. 字符是用单引号括起来的单个字符，`'a'`
    5. 英文字母-1个字节，汉字-3个字节
    6. 字符类型可以参与运算，相当于一个整数，运算时是按照码值运行

13. 布尔类型 bool

    1. 只允许取值 true 和 false
    2. bool类型占一个字节
    3. 适用于逻辑运算，一般用于程序流程控制 if for
    4. 不能用0或非0的整数来提到 false 和 true

14. 字符串类型 string

    1. 字符串就是一串固定长度的字符连接起来的字符系列，go的字符串是由单个字节连接起来的，go的字符串的字节使用utf-8编码标识unicode文本
    2. go中的字符串不可变
    3. 字符串的两种表示形式：
       - 双引号，会识别转义字符
       - 反引号，以字符串的原生形式输出，包括换行和特殊字符，可以防止攻击、输出源代码等效果
    4. 字符串的拼接：+，且 + 留在末尾可以换行继续写拼接

15. 基本数据类型的默认值：整型0、浮点型0、字符串""、布尔类型false

16. 基本数据类型的转换：golang中的数据类型 **不能自动转换**，否则无法通过编译，用起来跟python一样，比如 `var f float64 = float64(g)`

    - 既可以从 大->小，也可以从 小->大
    - 被转换的是值，而不是变量，变量本身的数据类型没有发生变化
    - 在转换时，如果往小的转，编译时不会报错，知识转换的结果是按溢出处理，不是直接裁剪，而是会变成一个奇怪的数（二进制的溢出处理）

17. 基本数据类型转string

    1. 推荐：`fmt.Sprintf("%参数", 表达式)`

       ```go
       var num1 int = 99
       var num2 float64 = 23.456
       var b bool = true
       var myCar byte = 'h'
       var str string
       
       str = fmt.Sprintf("%d", num1)
       fmt.Printf("str type %T str=%q \n", str, str)
       str = fmt.Sprintf("%f", num2)
       fmt.Printf("str type %T str=%q \n", str, str)
       str = fmt.Sprintf("%t", b)
       fmt.Printf("str type %T str=%q \n", str, str)
       str = fmt.Sprintf("%c", myCar)
       fmt.Printf("str type %T str=%q \n", str, str)
       ```

    2. 使用 strconv 包的函数，功能会更精细一点

       ```go
       str = strconv.FormatInt(int64(num1), 10)
       fmt.Printf("str type %T str=%q \n", str, str)
       str = strconv.Itoa(num1)
       fmt.Printf("str type %T str=%q \n", str, str)
       
       str = strconv.FormatFloat(num2, 'f', 10, 64)
       fmt.Printf("str type %T str=%q \n", str, str)
       str = strconv.FormatBool(b)
       fmt.Printf("str type %T str=%q \n", str, str)
       ```

18. string转基本数据类型：依然用 **strconv** 包

    - `	bb, _ = strconv.ParseBool(str2)`
    - `numm, _ = strconv.ParseInt(str2, 10, 64)`，把字符串转为十进制64位
    - `f1, _ = strconv.ParseFloat(str2, 64)`，把字符串转为64位的float
    - 注意：如果传入的str不是对应的数据，比如转整数给的却是字母，那么会直接转为0

19. **指针**

    1. 输出变量的地址：`fmt.Println("i 的地址是 ", &i)`

    2. 基本数据类型，变量存的是值，也叫值类型

    3. 指针类型，变量存的是一个地址，这个地址指向的空间存的才是值

       `var ptr *int = &num`，ptr是一个指针类型，ptr的类型是 `*int`，ptr本身的值是 `&i` 也就是i的地址

    4. 获取指针类型所指向的值，使用 `*ptr`

    5. 所有的值类型都有对应的指针类型，形式为：`*数据类型`；值类型包括

       - int系列
       - float系列
       - bool
       - string
       - 数组
       - 结构体 struct

       **变量直接存储值，内存通常在栈中分配**

    6. 引用类型：

       - 指针
       - 切片 slice
       - map
       - 管道 channel
       - 接口 interface

       **变量存储的是一个地址，这个地址对应的空间才真正存储数据（值），内存通常在堆上分配，当没有任何变量引用这个地址时，该地址对应的数据空间就成为一个垃圾，由gc来回收**

       PS：为什么上述都用通常，因为存在 **逃逸分析** ，并不绝对

    - 变量、方法、函数等命名称为标识符

    - 严格区分大小写，数字不能开头、不能包含空格、不能用系统保留关键字（25个），**数字+英文+_**

    - `_`：空标识符，可以代表任何其它标识符，但是值会被忽略，所以进作为占位符使用，**作为变量用不了**

    - PS：int、float32可以做标识符，但是不要这样用，不推荐（36个预定义标识符，包括基础函数类型和系统内嵌函数）

    - 注意事项：

      - 包名：保持package的名字和所在的目录保持一致，尽量有意义，不要跟已有的标准库冲突
      - 变量名、函数名、方法名：**驼峰法**，比如：stuName、goodPrice
      - :warning: 如果首字母大写，则可以被其他的包访问 **public**；如果首字母小写，则只能在本包中使用 **private**

23. 进制：

    1. 二进制：在golang中不能直接使用二进制来表示一个整数，它沿用了c的特点，输出二进制可以用 `%b` 输出
    2. 八进制（0110）
    3. 十进制
    4. 十六进制（0x110）：x不区分大小写

    :tractor: 进制的转换

    1. 其他进制转十进制：从最低位开始，每个位上的数字乘以【进制】的(位数-1)次方，再求和
    2. 十进制转其他进制：将该数不断除以【进制】，直到商为0为止，然后将每步得到的余数倒过来

    :tractor: 二进制转八进制、十六进制：将二进制数每 3(4) 位一组（从低位开始组合，高位补0），然后转换即可

    :tractor: 八进制、十六进制转二进制：将八（十六）进制的每1位，转成对应的一个3（4）位的二进制数即可

24. 位运算

    - 在计算机的内部，运行各种运算时，都是以二进制的方式来运行

    - 原码、反码、补码：对于有符号的而言

      - 二进制的最高位是符号位：0表示正数，1表示负数

      - 正数的原码、反码、补码都一样

        1的补码：0000 0001

      - 负数的反码 = 它的原码符号位不变，其它位取反

      - 负数的补码 = 它的反码 + 1

        -1的原码：1000 0001，反码：1111 1110，补码：1111 1111

        所以：1 - 1 = 0000 0001 + 1111 1111 = 0000 0000

      - 0的反码、补码都是0

      - **在计算机运算的时候，都是以 ==补码== 的方式来运行的**，因为计算机的世界里面没有补码

        ```
        -2^2
        
        -2 的原码：1000 0010
        -2 的反码：1111 1101
        -2 的补码：1111 1110
        2  的补码：0000 0010
        所以-2^2的补码：1111 1100
        所以-2^2的反码，就得-1，所以是 1111 1011
        所以-2^2的原码，就是符号位不变，其他的取反：
        1000 0100 -> -4
        ```

    - 移位运算

      - 右移 >> ：低位溢出，符号位不变，并用符号位补溢出的高位
      - 左移 << ：符号位不变，低位补0

## 4. 程序流程控制

1. 三大流程控制：顺序、分支、循环

2. 分支：

   - 单分支：:warning: golang要求必须带大括号，而不像有的语言，一行执行可以不带大括号，条件也不用写括号（:golf: 要求不写小括号，官方golang的代码也不写）

     :golf: go的if还有一个强大的地方就是条件判断语句里面允许生命一个变量，这个变量的作用域只能在该条件逻辑块内，其他地方就不起作用了

     ```go
     if 条件表达式 {
     	执行
     }
     
     if a := 1; a == 1 {
     	fmt.Println("a==1")
     } else {
     	fmt.Println("a!=1")
     }
     ```

   - 双分支

     `} else {` 要写在同一行

     ```go
     if 条件表达式 {
     	执行1
     }else{
     	执行2
     }
     
     if a := 1; a == 1 {
     	fmt.Println("a==1")
     } else {
     	fmt.Println("a!=1")
     }
     ```

   - 多分支，多分支只有一个执行入口，else不是必须的

     ```go
     if 条件表达式 {
     	执行1
     }else if{
     	执行2
     }
     ...
     else {
     	执行n
     }
     ```

3. switch

   - switch-case-default，**匹配项后面不需要再加 break 了**，default不是必须的

     ```go
     switch 表达式 {
     case 表达式1, 表达式2, ...:
     	语句块1
     case 表达式3, 表达式4, ...:
     	语句块2
     	...
     default:
     	语句块n
     }
     ```

   - switch和case后是一个表达式，常量、变量、一个有返回值的函数都行，但是对应值的数据类型要一致

   - case后面的表达式如果是常量，则不能重复（变量就识别不了啦，测了一下，如果重复了，匹配更前面的case）

   - switch可以不带表达式，当 if-else 来用 :question: 挺无语的，不是说好只有一种写法吗

     ```go
     var age int = 10
     switch {
     case age == 10:
     	fmt.Println("age = 10")
         fallthrough
     case age == 20:
     	fmt.Println("age = 20")
     default:
     	fmt.Println("age != 10, age != 20")
     }
     ```

   - switch 后也可以直接声明/定义一个变量，分号结束 :warning: 不推荐

     `switch grade := 90; {...}`

   - **switch 穿透** `fallthrough`，如果在case后面增加 fallthrough，则会继续执行下一个case。默认只穿透一层。【就跟其他语言的默认穿透一样了】 :warning: 不推荐

   - **Type Switch**：用于判断某个interface变量中实际指向的变量类型

     ```go
     var x interface{}
     var y = 10.0
     x = y
     switch i := x.(type) {
     case nil:
     	fmt.Println("x is nil")
     case int:
     	fmt.Println("x is int")
     case float64:
     	fmt.Println("x is float64")
     case func(int) float64:
     	fmt.Println("x is func(int) float64")
     case bool, string:
     	fmt.Println("x is bool or string")
     default:
     	fmt.Printf("未知类型 %T\n", i)
     }
     ```
     
   - switch和if的比较：如果判断的具体数值不多，用switch；其他就用if

4. 循环

   - 基本用法

     ```go
     for i := 1; i <= 10; i++ {
     	fmt.Println(i)
     }
     
     // 这种写法可以在外面使用i
     i := 1
     for ; i <= 10; i++ {
     	fmt.Println(i)
     }
     
     // 就是while了，配合break使用
     i := 1
     for i <= 10 {
         fmt.Println(i)
         i++
     }
     ```

   - for-range，方便的遍历字符串和数组 

     :warning: 用传统的for遍历无法遍历中文，会出现乱码，因为传统的遍历是按照字节来遍历的，而一个汉字在utf8是对应3个字节，需要将string转为rune切片 `str2 = []rune(str)`

     ```go
     str := "烟台欢迎你"
     for i, v := range str {
     	fmt.Printf("index=%d, val=%c\n", i, v)
     }
     ```

   - go语言中没有while和do-while的语法，前者是先判断再执行，后者是先执行再判断

   - 多重训话控制：实质上，嵌套循环就是把内层循环当成外层循环的循环体，建议一般使用2层 m*n 次

5. break

   - 用于中断当前for循环或者跳出switch语句

   - 随机数生成：

     - `rand.Seed(time.Now().Unix())`：返回一个从1970年1月1日0时0分0秒到现在的秒数
     - `rand.Seed(time.Now().UnixNano())`：相应的是纳秒
     - ``rand.Intn(100)+1`

   - break语句出现在多层嵌套的语句块中，可以通过标签指明要终止的是哪一层语句块 `break label2`

     ```go
     label2:
     	for i := 0; i < 10; i++ {
     		for j := 0; j < 10; j++ {
     			if j == 2 {
     				break label2
     				fmt.Println(j)
     			}
     		}
     	}
     ```

6. continue

   - 用于结束本次循环，继续执行下一次循环
   - continue语句出现在多层嵌套的语句块中，可以通过标签指明要跳过的是哪一层语句块 `continue label2`

7. goto

   - goto可以无条件的转移到指定行

   - 一般和if语句配合

   - :warning: 一般不主张使用goto语句，以免造成程序流程的混乱

     ```go
     	fmt.Println("ok1")
     	goto label2
     	fmt.Println("ok2")
     	fmt.Println("ok3")
     	fmt.Println("ok4")
     label2:
     	fmt.Println("ok5")
     	fmt.Println("ok6")
     ```

8. return：使用在方法或者函数中，表示跳出所在的方法或者函数

## 5. 函数和方法

1. 函数

   - 为完成某一个功能程序指令（语句）的集合，包括自定义函数和系统函数

     ```
     func 函数名 (形参列表) (返回值列表){
     	执行语句。。。
     	return 返回值列表
     }
     
     func cal(n1 float64, n2 float64, operator byte) (float64, int) {...}
     
     // 如果类型一样可以省略
     func cal(n1, n2 float32) float32 {}
     ```
     
   - 栈区：基本数据类型，一般分配到栈区（一般，是因为编译器存在逃逸分析）

   - 堆区：引用数据类型，一般分配到堆区，比如：切片、对象

   - 代码区：代码存放的位置

     ![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/5955c35007504caab71e1ac4f7b7f809.png)

2. 包

   - 包的本质就是创建不同的文件夹，来存放程序文件
   - go的每一个文件都是属于一个包，go是以包的形式来管理文件和项目目录结构的
   - 包的三大作用：
     - 区分相同名字的函数、变量等标识符
     - 当程序文件很多时，可以很好的管理项目
     - 控制函数、变量等访问范围，即作用域
   - 打包：`package xxx`
   - 引入包：`import "包的路径"`
   - 在给一个文件打包时，该包对应一个文件夹，文件的包名通常和文件所在的文件夹名一致，一般为小写字母
   - 先package，后import，import时不用写src
   - 需要跨包访问的函数，函数名的首字母需要大写，变量同样需要大写，同一个包下不能有同名的函数名
   - 给包取别名：`import util '包的路径'`
   - 如果要编译成一个可执行程序文件，就需要将这个包声明为main，即 `package main`

3. 函数的递归调用

   ![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/1eb2d1015b11462ba7c0752c4f5875e8.png)

   - 执行一个函数时，就创建一个新的受保护的独立空间（新函数栈）

   - 函数的局部变量是独立的，不会相互影响

   - 递归必须向退出递归的条件逼近，否则就是无限递归

   - 当一个函数执行完毕，或者遇到return，就会返回，遵守谁调用，就将结果返回给谁，同时当函数执行完毕或者返回时，该函数本身也会被系统销毁

   - 样例1：斐波那契数

     ```go
     func fib(n int) int {
     	if n <= 2 {
     		return 1
     	}
     	return fib(n-1) + fib(n-2)
     }
     ```

4. 函数注意事项和细节 🐹🐍

   1. 函数的形参列表可以是多个，返回值列表也可以是多个

   2. 形参列表和返回值列表的数据类型可以是值类型，也可以是引用类型

   3. 函数命名的首字母大写才能跨包，类似public，否则就是private

   4. 函数中的变量是局部的，在外面不生效

   5. **基本数据类型**和**数组**默认都是值传递，即进行值拷贝，在函数内修改，不影响原来的值

   6. 如果希望函数内的变量能够修改函数外的变量（默认以值传递方式的数据类型），可以传入变量的地址&，函数内以指针的方式操作变量，**从效果上看类似引用**

   7. go函数不支持函数重载（函数名相同，参数不同），类似的，python由于是动态语言，函数的参数类型在运行时才确定，也不支持函数重载，但是可以通过比如 `*args` 、`**kwargs` 、`functools.singledispatch` 等等来实现

   8. go中函数也是一种数据类型，可以赋值给一个变量，该变量就是一个函数类型的变量了，通过该变量可以对函数调用。因此函数可以作为形参，并调用

   9. 为了简化数据类型的定义，go支持自定义数据类型

      - `type myInt int`，虽然给int取了别名，但是go还是认为是两种不同的类型，使用还是需要显示转换
      - `type myfib func(int) int`，主要用来给函数类型取名

   10. :star: 支持对函数返回值命名，**命名返回值**（named return values）

       ```go
       func calc(a int, b int) (sum int, sub int) {
       	sum = a + b
       	sub = a - b
       	return
       }
       ```

   11. 可以使用占位符 `_` 来忽略返回值

   12. go支持可变参数 `func sum(args... int) sum int {}`，`func sum(n1 int, args... int) sum int{}`，形参列表的可变参数要是最后，其中args是slice切片，通过`args[index]` 可以访问到各个值

   13. 函数参数的传递方式

       - 值传递
       - 引用传递

       其实都是值传递，不同的是值的拷贝还是地址的拷贝，一般来说后者效率高，因为地址的数据量小

5. init 函数

   - 每一个源文件都可以包含一个init函数，该函数会在main函数执行前，被go运行框架调用

   - 如果一个文件中同时包含变量定义（全局变量）、init函数和main函数，则执行的流程是：变量定义 -> init函数 -> main函数

   - init函数最主要的作用，就是完成一些初始化的工作；如果是被其他的文件当做包引用，则会最先执行（:warning: 如果循环引用，会报错）

6. 匿名函数

   - 在定义匿名函数时就直接调用

   - 将匿名函数赋给一个变量（函数变量），再通过该变量来调用匿名函数 :bulb: 这下方便在函数里面定义函数啦


7. 闭包：

   - 介绍：一个函数与相关的引用环境组合的一个整体（实体），核心是**返回一个函数** :bulb: 所有的闭包都一样，**为了保存函数外的变量**

   - 闭包中的变量是保存的，匿名函数和外面的变量形成了一个整体

     ```go
     func AddUpper() func(int) int {
     	var n int = 10
     	return func(x int) int {
     		n = n + x
     		return n
     	}
     }
     
     f := AddUpper()
     fmt.Println(f(1)) // 11
     fmt.Println(f(2)) // 13
     ```

   - `strings.HasSuffix`：用来判断字符串是否包含后缀

     ```go
     func makeSuffix(suffix string) func(string) string {
     	return func(name string) string {
     		if !strings.HasSuffix(name, suffix) {
     			return name + suffix
     		}
     		return name
     	}
     }
     ```

     - 返回的匿名函数和suffix变量构成一个闭包
     - 如果使用闭包完成，好处是只需要把后缀传一次

8. defer

  - 为了在函数执行完毕后，及时的释放资源，go提供了defer延时机制

  - 当执行到defer时，会将defer后面的语句压入到独立的栈中，暂时不执行；**当函数执行完毕**后，再从defer栈，按照先入后出的方式出栈执行

    ```go
    func sum(n1 int, n2 int) int {
    	defer fmt.Println("ok n1=", n1)
    	defer fmt.Println("ok n2=", n2)
    
    	// 增加一句话，但是defer上的输出不变
    	n1++
    	n2++
    
    	res := n1 + n2
    	fmt.Println("ok3 res=", res)
    	return res
    }
    
    // ok3 res= 7 ——> 9
    // ok n2= 4
    // ok n1= 3
    ```

  - :warning: defer将语句放入到栈时，也会将相关的值拷贝同时入栈

  - defer最主要的价值是在函数执行完毕后，可以及时的释放函数创建的资源，比如关闭file、关闭connect

9. 变量作用域

   - 函数内部声明/定义的变量叫：局部变量，作用域仅限于函数内部
   - 函数外部声明/定义的变量叫：全局变量，作用域在整个包都有效，如果首字母为大写，则作用域在整个程序有效
   - 如果变量是在一个代码块，比如 for/if 中，那么这个变量的作用域就在该代码块
   - :warning: 类型推导 `:=` 只能用在函数内，不能用于声明跨包访问的变量

10. 常用的函数

    - **字符串**

      1. `len(字符串/数组)`：统计字符串的长度，按字节返回，中文会按3个来算
      2. `r := []rune(str)`：字符串遍历，同时处理有中文的问题，按字符而不是字节来遍历
      3. `n, err := strconv.Atoi("12")`：字符串转整数
      4. `str = strconv.Itoa(12123)`：整数转字符串
      5. `var bytes = []bytes("hello go")`：字符串转byte切片
      6. `str = string([]byte{97, 98, 99})`
      7. `str = strconv.Format(i int64, 进制 int)`：10进制转其他进制
      8. `strings.Contains(查找的字符串, 子字符串)`：查找子串是否在指定的字符串中
      9. `strings.Count(统计的字符串, 子字符串)`：统计一个字符串中有多少个指定的子串
      10. `strings.EqualFold(字符串1, 字符串2)`：不区分大小写的字符串比较，`==` 是区分字母大小写的
      11. `strings.Index(字符串, 子字符串)`：返回子串在字符串中第一次出现的index值，如果没有则返回-1
      12. `strings.LastIndex(字符串, 子字符串)`：返回子串在字符串中最后出现的index值，如果没有则返回-1
      13. `strings.Replace(字符串, 要替换的子字符串, 替换成的子字符串, 替换几个)`：将制定的子串替换成另一个子串，如果最后 n=-1 则表示全部替换
      14. `strings.Split(字符串, 子字符串)`：按照某个指定的字符串，为分割标识，将一个字符串拆分成字符串数组
      15. `strings.ToLower(字符串)`、`strings.ToUpper(字符串)`：将字符串的字母进行大小写转换
      16. `strings.TrimSpace(字符串)`：将字符串左右两边的空格去掉
      17. `strings.Trim(字符串, 子字符串)`：将字符串左右两边的子字符串去掉
      18. `strings.TrimLeft(字符串, 子字符串)`、`strings.TrimRight(字符串, 子字符串)`：将字符串左/右两边的子字符串去掉
      19. `strings.HasPrefix(字符串, 字符串)`、`strings.HasSuffix(字符串, 字符串)`：判断字符串是否以指定的字符串开头/结尾

    - **时间和日期函数**

      1. `time.Time`：数据类型，表示时间

## 6. 错误处理

   - 在默认情况下，当发生错误后（panic），程序就会退出（崩溃）

   - 如果我们希望，当发生错误后，可以捕获到错误，并进行处理，保证程序可以继续执行，甚至可以通知管理员

   - go语言追求简洁优雅，不支持传统的 `try...catch...finally`

   - go中引入的为：`defer，panic，recover`，go中可以抛出一个panic的异常，然后在defer中通过recover捕获这个异常，然后正常处理

      ```go
      func testException() {
      	defer func() {
      		err := recover()
      		if err != nil {
      			// 说明捕获到了异常
      			fmt.Println("err =", err)
      		}
      	}()
      	x1 := 10
      	x2 := 0
      	fmt.Println(x1 / x2)
      }
      ```

   - **自定义错误**

      - `errors.New("错误说明")`：返回一个error类型的值，表示一个错误
      - panic 内置函数，接受一个 inteface{} 类型的值作为参数，可以接收 error 类型的变量，输出错误信息，并退出程序

      > [!warning] java中错误是特殊的控制流，go中错误是普通的值、和返回结果平等
      >
      > | 对比项       | Java                                         | Go                                         |
      > | ------------ | -------------------------------------------- | ------------------------------------------ |
      > | 错误处理核心 | **异常对象 + 捕获机制**                      | **函数多返回值 + error 接口**              |
      > | 流程控制     | 抛出异常会**中断当前执行**，直接跳转到 catch | 默认**不中断正常流程**，错误和结果一起返回 |
      > | 显式 / 隐式  | 隐式传递（不用在参数里声明）                 | 显式传递（必须接收 / 处理返回值）          |
      > | 强制程度     | 受检异常**必须捕获 / 声明**，运行时异常可选  | 所有错误**都需要手动判断**，编译器不强制   |
      > | 崩溃处理     | Exception / Error                            | panic (严重错误) /recover (捕获崩溃)       |
      >
      > ### Java
      >
      > - 所有严重错误都继承自 `Error`/`Exception`
      > - 空指针、数组越界：`RuntimeException`
      > - 崩溃统一用 `catch` 捕获
      >
      > ### Go
      >
      > - 普通错误：`error` 接口（业务错误，必须处理）
      > - 严重崩溃：`panic()`（空指针、越界、程序无法继续）
      > - 捕获崩溃：`recover()`（只能在 `defer` 中使用，类似 catch，但**极少使用**）
      >
      > Go 官方建议：**panic 只用于真正的致命错误，不要用它代替 error**。

- **排序和查找**

  - 分类：

    - 内部排序：所有数据都加载到内存后排序，有：交换式排序法（冒泡、快排）、选择式排序法、插入式排序法等
    - 外部排序：数据量过大，无法全部加载到内存，需要借助外部存储进行排序，有：合并排序法、直接合并排序法等

  - 冒泡排序

    - 让前面的数跟后面的数比较，如果前面的数大，就交换，所以第一轮就把最大的冒泡到了最后一个位置

    - 一共经过 `arr.length-1` 次轮数的比较；每一轮的比较次数会逐渐较少，第一次比较次数为 `arr.length-1`

      ```go
      func BubbleSort(arr *[5]int) {
      	fmt.Println("排序前：", *arr)
      	for i := 0; i < len(*arr)-1; i++ {
      		for j := 0; j < len(*arr)-1-i; j++ {
      			if (*arr)[j] > (*arr)[j+1] {
      				(*arr)[j], (*arr)[j+1] = (*arr)[j+1], (*arr)[j]
      			}
      		}
      		fmt.Printf("第 %v 轮排序：%v\n", i+1, *arr)
      	}
      }
      ```

  - 顺序查找：index=-1，如果找到了，把index赋值，并break

  - 二分查找（该数组要求有序）

    - 中间下标：（左下标+右下标）/2，比较中间下标的值与要找的值比较
    - 如果 arr[middle] > finalVal，则右下标 = middle - 1
    - 如果 arr[middle] < finalVal，则左下标 = middle + 1
    - 如果 arr[middle] = finalVal，找到
    - 结束标志： leftIndex > rightIndex

    ```go
    func BinaryFind(arr *[6]int, findVal int, leftIndex int, rightIndex int) {
    	if leftIndex > rightIndex {
    		fmt.Println("没有找到")
    		return
    	}
    
    	middle := (leftIndex + rightIndex) / 2
    	if (*arr)[middle] > findVal {
    		fmt.Printf("左边界：%v，右边界：%v\n", leftIndex, middle-1)
    		BinaryFind(arr, findVal, leftIndex, middle-1)
    	} else if (*arr)[middle] < findVal {
    		fmt.Printf("左边界：%v，右边界：%v\n", leftIndex, middle-1)
    		BinaryFind(arr, findVal, middle+1, rightIndex)
    	} else {
    		fmt.Println("找到，索引是：", middle)
    	}
    }
    ```

## 8. map

- 就是键值对、字典
- `var mm map[string]int`，key选的最多的是string、int，key不能重复，否则会覆盖；value基本跟key一样

- 获取长度：`len(map)`
- map的切片：`ls := []map[string]string`，这不就是 `List<Map>` 吗
- map的排序：没有一个专门针对map的kay排序的方法，再加上map是无序的
  - `sort.Ints(keys)`：对 keys 的切片进行排序
  - 然后遍历排序后的 keys，在获取完成排序的值
- map的使用细节
  - map是引用类型，遵循引用类型传递的机制
  - map的容量达到后，会自动扩容，并不会发生panic
  - map的value也经常使用struct结构体，更适合管理复杂的数据


## 9. 面向对象

- go语言中没有类的概念，用 **结构体struct** 来代替
  - golang支持面向对象编程（OOP）特性
  - golang去掉了OOP语言的继承、方法重载、构造函数、析构函数、隐藏的this指针等
  - golang仍然有 **继承、封装和多态** 的特性，虽然没有extends关键字，但是通过匿名字段实现
  - golang OOP很优雅，是 **面向接口编程**，interface，耦合性地

- 结构体是自定义的数据类型，代表一类事物，结构体变量是具体的实例

- 结构体在内存中的布局，结构体是 **值类型**

  ![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/b5fb6e4d6b7046dca55d88f4adf16ad5.png)

- 结构体声明

  ```go
  type 结构体名称 struct {
  	filed1 string
  	Filed2 int
  	filed3 float32
  }
  ```

  结构体名称首字母大写、结构体属性首字母大写决定了是否可以被外部使用

- 注意：

  - 字段声明的语法跟变量是一样的
  - 字段的类型可以是是基本类型、数组和引用类型，所以没赋值前，也是默认值，指针、slice和map是nil，需要先new/make再赋值，指针是直接给nil，slice是[]，map是map[]，其实都是nil
  - 不同的结构体的变量是独立的，不会相互影响，因为结构体是值类型

- 创建结构体变量和访问结构体字段

  - `var p Person`

  - `var p Person = Person{"mary", 20}`

  - `var p *Person = new(Person)`，获得的是指针，需要 `*p` 去赋值和取值

    :warning: go为了使用方便，p直接赋值和取值也行，底层会自动 `*p`

  - `var p *Person = &Person{}`，因为p是指针，因此标准的访问字段的方法是 `(*p).name` 去取值赋值，同上，也可以简化`*`

  - `.` 的优先级高，所以 `*p` 要包括号

- 结构体的所有字段在内存里是连续的，方便快速寻址；如果结构体指针是字段的话，地址连续，但是指向的地址不一定连续

- 结构体的强转要求字段名称、类型、个数一致

- 结构体进行type重新定义（相当于取别名），golang认为是新的数据类型，但是相互之间可以强转 

  :warning: 给类型取别名也是一样的 `type integer int`

- struct的每个字段上可以写tag，这个tag可以通过反射机制获取，常用于序列化和反序列化 **Name string \`json:"name"\`**

  - 序列化：struct - json，`json.Marshal(sturct变量)`
  - 反序列化：json - struct
  - 加tag可以把属性写小写，这样json的小写来了之后也能识别

- **方法**

  - 方法是作用在指定的数据类型上的，自定义类型都可以有方法，不仅仅是struct

    ```go
    type A struct {
    	Num int
    }
    
    func (this A) test() {
    	fmt.Println("test")
    }
    ```

    :bulb: 数据和行为可以分离，通过清晰的接口"通信"，而不是硬绑在一起。

  - 方法调用时，会把调用方法的变量（也就是struct的示例）当做实参也传递给方法**（值传递，复制一份）**

  - 在更多的情况下，写方法时不会用值拷贝，而是地址拷贝的方式：`func (this *A) test()`，同时，由于语法糖的原因，执行语句不需要额外的 `(*a).属性`，外部调用时也不需要额外的 `(&c).test()`

  - 方法的访问范围控制的规则与函数的一致，注意大小写，来控制是否对外访问

  - 如果一个类型实现了 `String()` 这个方法，那么 `fmt.Println` 默认会调用这个变量的 `String()` 进行输出

- 方法和函数的区别

  - 调用方式：函数是 函数名(实参列表)；方法是 变量.方法名(实参列表)
  - 对于普通函数，参数为值类型时，不能将指针类型的数据直接传递，如果是值类型，同样的也不能传指针类型
  - 对于方法，接收者为值类型时，可以直接用指针类型的变量调用方法，反之亦然。:warning: 这句话的意思说的是，`func (p Person) test(){}` 这里的 `p Person` 可以跟 `p *Person` 互换，因为底层做了优化的。**但是！** 实际的变量行为是有区别的，值传递就不会影响主函数的对象，指针传递就会影响，golang做的优化只是单纯简化了 p、*p、&p

- 结构体创建实例

  - `var stu Student = Student{"tom". 10}`，即，`stu := Student{"tom~", 10}`，还可以指定字段，这样就不依赖字段的定义顺序了：`stu := Student{Name: "mary", Age: 20,}`
  - 获取结构体的指针：`var stu Student= &Student{"tom~", 10}` 等价于 `var stu *Student= &Student{"tom~", 10}`，类型推导或者指定字段也是一样的

- golang没有构造函数，通常可以使用 **简单工厂模式** 来解决

  ```go
  func NewStudent(n string, s float64) *student {
  	return &student{
  		Name: n,
  		Score: s,
  	}
  }
  ```

  这样好处是，外部可以不直接访问 student 结构体，通过函数拿到新建 student 的指针；类似的可以通过写 set/get 方法避免直接访问结构体的属性（比如，属性小写，方法大写）

- OOP思想

  1. **抽象**：struct

  2. **封装**：encapsulation，就是把抽象出来的字段和字段的操作，封装在一起，数据被保护在内部，授权才能操作

     优势：隐藏实现细节；对数据进行验证

     实现：对结构体中的属性进行封装；通过方法、包实现封装

  3. **继承**：解决代码复用、更接近人类思维

     通过嵌入匿名结构体的方式实现

     ```go
     type Goods struct {
     	Name string
     	Price int
     }
     
     type Book struct{
     	Goods
     	Write string
     }
     
     // 主函数中使用
     b := &Book{}
     b.Goods.Name = "tom"
     b.Goods.Price = 8
     b.testing()
     b.Goods.SetScore(77)
     ```

     - 继承的深入讨论

       - 结构体继承后，可以使用嵌套匿名结构体所有的字段和方法，包括大小写

       - 匿名结构体字段访问可以简化：`b.A.say()` -> `b.say()`，但是：当有相同字段或者方法的时候，**就近原则**，如果想访问匿名结构体的话，还是用不省略的方案

       - 如果结构体嵌入两个或者多个结构体，如果两个匿名结构体有同名字段和方法、同时结构体本身没有同名的字段和方法，则必须**指定匿名结构体的名字**，否则编译报错

       - 如果一个struct嵌套了一个有名结构体，这种模式就叫 **组合** 就不是 **继承** 了，这种情况下，访问组合的结构体的字段和方法时，必**须带上结构体的名字**

         ```go
         type A struct {
         	Name string
         	Age int
         }
         
         type B struct {
         	a A
         }
         ```

       - 嵌套匿名结构体后，可以在创建结构体变量时，直接指定各个匿名结构体字段的值：`tv := TV{ Goods{Name: "v-01", Price: 5999}, Brand{"haier", "shandong"} }`

       - 嵌套匿名结构体的时候，可以嵌套匿名结构体的 指针类型 `*A`，初始化的时候记得 `&B{...}`，用的时候 `*tv.B` 

         :bulb: 一般不用指针继承或组合

     - 多重继承：一个struct嵌套了多个匿名结构体，建议不要使用多重继承

  4. **接口 interface**

     - golang中的多态主要是通过接口来实现的

     - 接口类型可以定义一组方法（没有方法体），但不需要实现，且接口中不能包含任何变量

     - 实现了接口中所有方法，才叫实现了接口

     - **多态和高内聚低耦合的思想**

     - golang中不需要显示的 implements 接口，因此没有类似的关键字

       > 🦆 **鸭子类型（Duck Typing）**
       >
       > “如果它走起路来像鸭子，叫起来也像鸭子，那它就是鸭子。”
       >
       > 在编程中，**鸭子类型** 是一种**动态类型**的特性，**不关心对象的类型**，而是**关心对象是否具有所需的行为（方法或属性）**。

       ```go
       func main(){
       	computer := Computer{}
       	phone := Phone{}
       	camera := Camera{}
       
       	computer.BindingUsb(phone)
       	computer.BindingUsb(camera)
       }
       
       type Usb interface {
       	Start()
       	Stop()
       }
       
       type Phone struct {
       }
       
       func (phone Phone) Start() {
       	fmt.Println("手机开始工作")
       }
       
       func (phone Phone) Stop() {
       	fmt.Println("手机停止工作")
       }
       
       type Camera struct {
       }
       
       func (camera Camera) Start() {
       	fmt.Println("相机开始工作")
       }
       
       func (camera Camera) Stop() {
       	fmt.Println("相机停止工作")
       }
       
       type Computer struct {
       }
       
       func (computer Computer) BindingUsb(usb Usb) {
       	usb.Start()
       	usb.Stop()
       }
       ```

     - 注意事项

       - 接口本身不能创建实例，但可以指向一个实现了该接口的自定义类型变量；一个自定义类型只有实现了某个接口，才能将该自定义类型的实例赋给接口类型

         :bulb: 用来实现多态

       - 只要是自定义数据类型，就可以实现接口，不仅仅是结构体类型

         比如 `type integer int`，类似起别名，但是直接给 `func (i integer) Say(){...}` 实现方法，那么integer就实现了对应的接口，从而 `var i Ainterface = 10`，可以赋给接口类型

       - 一个自定义类型可以实现多个接口

       - golang接口中不能有任何变量

       - 接口可以继承接口，实现的话也要实现继承的接口的所有方法

       - interface类型默认是一个指针，**引用类型**，如果没有对interface做初始化就使用，那么会输出nil

       - 空接口 `interface{}` 没有任何方法，所以所有类型都实现了空接口，**可以把任何一个变量赋给空接口**

     - 接口vs继承

       - 接口是对继承的补充
       - 继承的价值：解决代码的复用性和可维护性
       - 接口的价值：设计规范
       - 接口比继承更灵活，like-a与is-a的关系
       - 接口可以实现代码解耦

  5. **多态**

     - 多态的特征是通过接口实现的，可以按照统一的接口来调用不同的实现。**封装、继承、多态：面向对象的三大特征**
     - **多态参数**：可以接受不同的变量，因为这些变量的struct实现了接口的所有方法
     - **多态数组**：可以放不同struct变量的数组

- 类型断言

  - `b = a.(Point)`：判断a是否是指向Point类型的变量，是就转换并赋值，否则就报错

  - 解决空类型 `interface{}` 的赋值问题

  - 检查断言是否成功：`b, ok = a.(Point)`

  - 简洁写法：`if b, ok = a.(Point); ok{...}else{...}`

  - 断言场景1：多态数组中，区分不同的类型（用if），调用不同的方法

  - 断言场景2：传入参数的类型判断 `a.(type)`，这里的type是个关键字，固定写法

    ```go

    case float64:...
    case int, int32, int64:...

    case string:...
    default:...
    ```

## 12. goroutine协程和channel管道

1. 进程和线程

   - 进程是程序在操作系统中的一次执行过程，是系统进行资源分配和调度的基本单位
   - 线程是进程的一个执行实例，是程序执行的最早单元，它是比进程更小的能独立运行的基本单位
   - 一个进程可以创建和销毁多个线程，同一个进程中的多个线程可以并发执行
   - 一个程序至少有一个进程，一个进程至少有一个线程

2. 并发和并行

   - 多线程程序在单核上运行，并发
   - 多线程程序在多核上运行，并行

3. go的协程和go的主线程

   - go的主线程：有叫线程也又叫进程的，一个go线程上可以起多个协程，**协程就是轻量级的线程**。
   - go协程的特点：
     - 有独立的栈空间
     - 共享程序堆空间
     - 调度由用户控制
     - 协程式轻量级的线程
   - java出的虚拟线程对标的就是协程
   - golang的协程机制是重要的特点，可以轻松的开启上万个协程。其它变成语言的并发机制一般是基于线程的，开启过多的线程，资源耗费大，这里就凸显了golang在并发上的优势了

4. 协程代码样例

   - 如果主线程退出了，而协程即使还没有执行完毕，也会退出

     	var i int64 = 1

9. 为什么需要 channel

   1. 主线程在等待所有goroutine全部完成的时间很难确定，设置10s仅仅是估算
   2. 如果主线程休眠时间长了，会加长等待时间，如果等待时间短了，可能还有goroutine处于工作状态，这时也会随主线程的退出而销毁
   3. 通过全局变量加锁同步来实现通讯，也并不利于多个协程对全局变量的读写操作

10. channel的介绍

    1. channel的本质就是一个数据结构 —— **队列**

    2. 数据是先进先出，FIFO

    3. 线程安全，多goroutine访问时，不需要加锁

    4. channel有类型，一个string的channel只能存放string类型，如果想放多种类型，可以把channel声明为空接口（不建议）

       `var allChan chan interface{}`，但是使用的时候需要用类型断言：`xxx.(Cat)`

       ```go
       allChan := make(chan interface{}, 10)
       allChan <- 10
       allChan <- "hello golang"
       allChan <- Cat{"tom", 10}
       
       <-allChan
       <-allChan
       newCat := <-allChan
       fmt.Printf("type is %T, value is %v\n", newCat, newCat)
       fmt.Println(newCat.(Cat).Name)
       ```

    5. 放满之后不能再放，取空之后不能再取

    6. 取的时候可以不接受

    ```go
    	go readData(intChan, exitChan)
    	<-exitChan
    
    }

    		intChan <- i

    }

    	for {
    		v, ok := <-intChan
    		if !ok {
    			fmt.Println("管道已关闭")
    			break
    		}
    		fmt.Println("从管道中读到的数据是：", v)
    	}
    	exitChan <- true
    	close(exitChan)
    }
    ```

## 13. 反射

1. 基本介绍

   - 反射可以在运行时动态获取变量的各种信息，比如类型type、类别kind

   - 如果是结构体变量，还可以获取到结构体本身的信息，包含字段、方法

   - 通过反射，可以修改变量的值，可以调用关联的方法

   - 使用反射：`import "reflect"`

     - `reflect.TypeOf`：是一个接口 interface
     - `reflect.ValueOf`：是一个结构体 struct

   - 反射中的变量、`interface{}` 和 `reflect.Value` 是可以相互转换的

     - 将 `Student` 变量转为 `interface{}`：传递参数即可
     - 将 `interface{}` 转为 `reflect.Value`：`rVal = reflect.ValueOf(b)`
     - 将 `reflect.Value` 转为 `interface{}`：`iVal = rVal.Interface()`
     - 将 `interface{}` 转为原来的变量类型：`v := iVal.(Student)`，使用断言

     ![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/4f97277c62294e019e84a60688eceea5.png)

2. 反射的应用场景

   1. 不知道接口调用哪个函数，根据传入参数在运行时确定调用的具体接口
   2. 对结构体序列化时，制定tag，生成对应的字符串

3. 反射的快速入门

   ```go
   func main() {
   	var num int = 100
   	reflectTest01(num)
   }
   func reflectTest01(b interface{}) {
   	rType := reflect.TypeOf(b)
   	fmt.Println(rType, rType.Name())
   	rVal := reflect.ValueOf(b)
   	fmt.Println(rVal, rVal.Int())
   
   	iV := rVal.Interface()
   	num2 := iV.(int)
   	fmt.Println("num2=", num2)
   }
   ```

4. 反射的本质是 **运行时** 的，在编译阶段没法完成类型的识别，因此需要将 `interface{}` 通过断言转换成需要的类型

   - 普通数据类型直接用 `reflect.ValueOf(b).Int()`，如果硬要 `.Float()` 就会在运行时报错 panic
   - 如果是 struct 数据，就只能用断言的了 `reflect.ValueOf(b).(Student)`

   ```go
   func main() {
   	stu := Student{Name: "tom", Age: 18}
   	reflectTest02(stu)
   }
   
   type Student struct {
   	Name string
   	Age  int
   }
   
   func reflectTest02(b any) {
   	rType := reflect.TypeOf(b)
   	fmt.Println(rType, rType.Name())
   	rVal := reflect.ValueOf(b)
   
   	iV := rVal.Interface()
   	fmt.Printf("iV=%v, iV Type=%T \n", iV, iV)
   	stu, ok := iV.(Student)
   	if ok {
   		fmt.Println("stu.Name=", stu.Name)
   	}
   }
   ```

5. `reflect.Value.Kind`：获取变量的类别，返回的是一个常量

   ```go
   kind1 := rType.Kind()
   kind2 := rVal.Kind()
   fmt.Println("kind1=", kind1, "kind2=", kind2)
   
   # 结果是一样的
   ```

   Type是类型、Kind是类别，Type和Kind可能是相同的，也可能是不同的

   - 比如 `var num int = 10` ，都是 `int`
   - 比如 `var stu Student`，的 Type 是 `包名.Studeng`，Kind 是 `struct`

6. 通过反射来修改变量，使用 `SetXxx` 方法来设置，需要通过对应的 **指针** 类型来完成，这样才能改变传入的变量的值，同时需要使用到 `reflect.Value.Elem()` 方法

   ```go
   func main() {
   	var num int = 11
   	reflectTest03(&num)
   	fmt.Println("num=", num)
   }
   
   func reflectTest03(b interface{}) {
   	rVal := reflect.ValueOf(b)
   	// 类别是 ptr，指针
   	fmt.Println("rVal kind=", rVal.Kind())
   	// 需要先拿到指针指向的元素，再赋值
   	rVal.Elem().SetInt(100)
   }
   ```

8. 反射最佳实例：使用反射来遍历结构体的字段，调用结构体的方法，并获取结构体标签的值

   ```go
   func main(){
   	var a MM = MM{
   		Name:  "牛魔王",
   		Age:   500,
   		Score: 100.0,
   	}
   	TestStruct(a)
   
   }
   
   type MM struct {
   	Name  string `json:"name"`
   	Age   int    `json:"monster_age"`
   	Score float32
   	Sex   string
   }
   
   func (s MM) Print() {
   	fmt.Println("--- start ---")
   	fmt.Println(s)
   	fmt.Println("--- end ---")
   }
   
   func (s MM) GetSum(n1, n2 int) int {
   	return n1 + n2
   }
   
   func (s MM) Set(name string, age int, score float32, sex string) {
   	s.Name = name
   	s.Age = age
   	s.Score = score
   	s.Sex = sex
   }
   
   func TestStruct(a interface{}) {
   	// 获取 reflect.Type 类型
   	typ := reflect.TypeOf(a)
   	// 获取 reflect.Value 类型
   	val := reflect.ValueOf(a)
   	// 获取变量的类别
   	kd := val.Kind()
   	// 如果传入的不是结构体，我就不玩了
   	if kd != reflect.Struct {
   		fmt.Println("传入的参数必须是结构体")
   		return
   	}
   	// 获取到该结构体有几个字段
   	num := val.NumField()
   	fmt.Println("结构体的字段数量：", num)
   	// 遍历结构体的字段
   	for i := 0; i < num; i++ {
   		fmt.Printf("第%d个字段，值是%v\n", i, val.Field(i))
   		tagVal := typ.Field(i).Tag.Get("json")
   		// 如果该字段有tag标签就显示
   		if tagVal != "" {
   			fmt.Printf("tag是%s\n", tagVal)
   		}
   	}
   	// 获取到该结构体有多少个方法
   	numOfMethod := val.NumMethod()
   	fmt.Println("结构体的方法数量：", numOfMethod)
   	// 方法的顺序跟编写的顺序无关，跟方法名的排序有关（ASCII码）
   	val.Method(1).Call(nil)
   
   	// 调用结构体的第1个方法 Method(0)
   	var params []reflect.Value
   	params = append(params, reflect.ValueOf(10))
   	params = append(params, reflect.ValueOf(40))
   	// 传入的是 []reflect.Value 切片，返回的结果也是
   	res := val.Method(0).Call(params)
   	fmt.Println("res=", res[0].Int())
   }
   ```

     ```go
     		}
     	}
     }
     ```

     ```go
     package main
     
     import (
     	"fmt"
     	"io"
     
     	// 做socket网络开发时，net包含以后我们需要所有的方法和函数
     	"net"
     )
     
     func process(conn net.Conn) {
     	// 这里我们循环的接收客户端发送的数据
     	defer conn.Close()
     	for {
     		// 创建一个新的切片
     		buf := make([]byte, 1024)
     		// 等待客户端通过conn发送信息流，如果客户端没有Write，那么会一直阻塞
     		// fmt.Println("服务器等待客户端发送数据。。。", conn.RemoteAddr().String())
     		n, err := conn.Read(buf)
     		if err == io.EOF {
     			fmt.Println("客户端退出了。。。")
     			return
     		}
     		// 显示客户端发送的内容到服务器的终端
     		fmt.Println("服务器收到客户端发来的数据：", string(buf[:n]))
     	}
     }
     
     func main() {
     	fmt.Println("服务器开始监听了。。。")
     	listen, err := net.Listen("tcp", "0.0.0.0:8888")
     
     	if err != nil {
     		fmt.Println("net.Listen err=", err)
     		return
     	}
     	defer listen.Close() // 延时关闭
     
     	for {
     		fmt.Println("等待客户端连接。。。")
     		conn, err := listen.Accept()
     		if err != nil {
     			fmt.Println("listen.Accept err=", err)
     			continue
     		} else {
     			fmt.Println("客户端连接成功。。。", conn, "客户端IP：", conn.RemoteAddr().String())
     		}
     		go process(conn)
     	}
     }
     ```

   - redis连接池

      - 事先初始化一定数量的连接，放入到连接池
      - 当go需要操作Redis时，直接从Redis连接池取出连接即可
      - 这样可以节省临时获取Redis连接的实践，从而提高效率

      ```go
      package main
      
      import (
      	"fmt"
      
      	"github.com/gomodule/redigo/redis"
      )
      
      var pool *redis.Pool
      
      func init() {
      	pool = &redis.Pool{
      		// 最大空闲链接数
      		MaxIdle: 8,
      		// 表示和数据的最大连接数，0表示没有限制
      		MaxActive: 0,
      		// 最大空闲时间
      		IdleTimeout: 300,
      		// 初始化连接池，链接哪个ip的redis
      		Dial: func() (redis.Conn, error) {
      			return redis.Dial("tcp", "127.0.0.1:6379")
      		},
      	}
      }
      
      func main() {
      	conn := pool.Get()
      	defer conn.Close()
      
      	_, err := conn.Do("SET", "name", "张三")
      	if err != nil {
      		fmt.Println("set failed, err:", err)
      		return
      	}
      
      	r, err := redis.String(conn.Do("GET", "name"))
      	if err != nil {
      		fmt.Println("get failed, err:", err)
      	}
      	fmt.Println("name:", r)
      }
      ```

2. **稀疏数组** sparse array

   - 记录数组一共有几行几列，有多少个不同的值
   - 把不同元素的行、列、值，记录在一个小数组中

   ```go
   package main
   
   import "fmt"
   
   type ValNode struct {
   	row int
   	col int
   	val int
   }
   
   func main() {
   	var chessMap [11][11]int
   	chessMap[1][2] = 1
   	chessMap[2][3] = 2
   
   	for _, v := range chessMap {
   		for _, v2 := range v {
   			// fmt.Printf("第%d行, 第%d列, 值为%d\n", i, j, v2)
   			fmt.Printf("%d\t", v2)
   		}
   		fmt.Println()
   	}
   
   	// 转成稀疏数组
   	var sparseArr []ValNode
   	sparseArr = append(sparseArr, ValNode{11, 11, 0})
   	for i, v := range chessMap {
   		for j, v2 := range v {
   			if v2 != 0 {
   				sparseArr = append(sparseArr, ValNode{i, j, v2})
   			}
   		}
   		fmt.Println()
   	}
   
   	// 输出稀疏数组
   	fmt.Println("稀疏数组为:")
   	for i, valNode := range sparseArr {
   		fmt.Printf("第%d行, row:%d, col:%d, val:%d\n", i, valNode.row, valNode.col, valNode.val)
   	}
   
   	// 恢复数组
   	var chessMap2 [11][11]int
   	for _, valNode := range sparseArr[1:] {
   		chessMap2[valNode.row][valNode.col] = valNode.val
   	}
   	fmt.Println("恢复后的数组为:")
   	for _, v := range chessMap2 {
   		for _, v2 := range v {
   			fmt.Printf("%d\t", v2)
   		}
   		fmt.Println()
   	}
   }
   ```

3. **队列** queue

   - 有序列表，可以用数组、链表来实现，先入先出

   - 普通队列：缺点是没有有效利用数组空间

     ```go
     package main
     
     import (
     	"errors"
     	"fmt"
     )
     
     type Queue struct {
     	maxSize int
     	array   [5]int
     	front   int // 不含队首
     	rear    int // 含队尾
     }
     
     // 添加队列
     func (this *Queue) addQueue(val int) (err error) {
     	// 先判断队列是否已满
     	if this.rear == this.maxSize-1 {
     		// 含最后一个元素
     		return errors.New("队列已满")
     	}
     
     	this.rear++
     	this.array[this.rear] = val
     	return
     }
     
     // 显示队列
     func (this *Queue) showQueue() {
     	if this.front == this.rear {
     		fmt.Println("队列为空")
     		return
     	}
     
     	for i := this.front + 1; i <= this.rear; i++ {
     		fmt.Printf("arr[%d]=%d\n", i, this.array[i])
     	}
     }
     
     // 取出数据
     func (this *Queue) getQueue() (val int, err error) {
     	if this.front == this.rear {
     		return -1, errors.New("队列为空")
     	}
     
     	this.front++
     	val = this.array[this.front]
     	return val, nil
     }
     
     func main() {
     	queue := &Queue{maxSize: 5, front: -1, rear: -1}
     	for i := 0; i < 5; i++ {
     		err := queue.addQueue(i)
     		if err != nil {
     			fmt.Println(err.Error())
     			break
     		}
     	}
     	queue.showQueue()
     	err := queue.addQueue(6)
     	if err != nil {
     		fmt.Println(err.Error())
     	}
     
     	for i := 0; i < 5; i++ {
     		val, err := queue.getQueue()
     		if err != nil {
     			fmt.Println(err.Error())
     			break
     		}
     		fmt.Println("取出的元素是：", val)
     	}
     	val, err := queue.getQueue()
     	if err != nil {
     		fmt.Println(err.Error())
     	} else {
     		fmt.Println("取出的元素是：", val)
     	}
     }
     ```

   - 环形队列

     - 什么时候表示队列满：`(tail + 1) % maxSize == head`
     - 什么时候表示队列空：`tail == head`
     - 初始化时：`tail = head = 0`，跟非环形队列的-1是不同的
     - 怎么统计该队列有多少个元素：`(tail + maxSize - head) % maxSize`
